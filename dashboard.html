How To Use This Dashboard

A quick orientation for getting value out of the hackingtool plugin.

🚀 Getting Started

STEP 1
Pick a category

Start in Overview, filter by domain (Web, Wireless, Forensics…), and click in.

STEP 2
Find a tool

Use the Detailed List search to match tool names or descriptions instantly.

STEP 3
Grab a prompt

Copy a best-practice prompt and adapt the scope to your authorized target.

STEP 4
Execute & document

Run within scope, capture evidence, and feed results back into reporting.

🧭 Workflow Tips

  • Chain categories in order: Information Gathering → Vuln Analysis → Exploitation → Post-Exploitation → Reporting.
  • Keep a scoped targets.txt and reuse it across recon tools for consistency.
  • Prefer passive recon first; escalate to active scanning only when authorized.
  • Snapshot tool output to timestamped files so your report timeline is defensible.
Tip: Run this dashboard locally (open dashboard.html) or publish via GitHub Pages — it's a single file with zero dependencies and no build step.

⚖️ Authorization & Ethics

These tools are for authorized security testing, CTFs, research, and defense only.

Always required: written permission / a signed scope before testing any system you do not own. Unauthorized access is illegal in most jurisdictions. Stay inside scope, respect rules of engagement, and disclose findings responsibly.
